# セキュリティポリシー

## サポート対象バージョン

| バージョン | サポート状況 |
|-----------|------------|
| 3.0.x     | ✅ サポート中 |
| < 3.0     | ❌ サポート終了 |

## 脆弱性の報告

セキュリティ上の脆弱性を発見された場合は、**公開 Issue を作成せず**、以下の方法でご報告ください。

### 報告方法

1. **GitHub Security Advisory** （推奨）:
   [Security Advisory を作成](https://github.com/TadFuji/ceremonial-rescue-jp/security/advisories/new) してください。

2. **非公開 Issue**:
   リポジトリの Security タブから非公開で報告できます。

### 報告に含めてほしい情報

- 脆弱性の概要
- 再現手順
- 影響を受けるファイル・バージョン
- 可能であれば修正案

### 対応プロセス

1. **確認**: 報告受領後、3営業日以内に確認の連絡をいたします
2. **調査**: 脆弱性の影響範囲を評価します
3. **修正**: パッチを作成し、テストを実施します
4. **公開**: 修正完了後、セキュリティアドバイザリとして公開します

### 対象範囲

このプロジェクトは Agent Skill（AIアシスタント向けの知識・スクリプト集）です。以下が報告の対象です:

- Python スクリプト (`scripts/`) のコードインジェクション脆弱性
- テンプレート (`assets/templates/`) 内の意図しない情報漏洩
- 参照データ (`references/`) の不正確な情報（文化的に不適切な内容を含む）

### 対象外

- AI プラットフォーム（Claude, OpenClaw 等）自体の脆弱性
- 一般的なマナー情報の正確性に関する議論（→ Issue をご利用ください）

ご報告いただいた方には、修正リリース時にクレジットを記載いたします（ご希望の場合）。
